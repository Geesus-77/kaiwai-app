# 👑 KAIWAI Project Master Rules (헌법)
이 문서는 KAIWAI 프로젝트의 핵심 규범이다. 모든 코드 생성/수정 전 이 문서를 먼저 읽고 준수해야 한다.

## 1. 아키텍처 원칙
- **Single File Mandate:** UI(HTML), 로직(JS), 스타일(Tailwind CSS)은 `index.html` 단일 파일에 작성한다.
- **BaaS 전략:** Supabase를 주력으로 사용하되, 모든 로직은 클라이언트 렌더링이 아닌 DB/RPC 수준에서 무결성을 보장한다.
- **디자인 시스템:** 둥근 모서리, 글래스모피즘, 핑크/화이트 톤의 감성 디자인 유지.

## 2. 보안(Security) 원칙 (절대 타협 불가)
- **Zero Trust:** 클라이언트가 보내는 모든 금액/수량/상태 변경값은 맹신하지 않는다.
- **서버 사이드 검증:** RPC 내부에 IF 문을 사용하여 비즈니스 로직(환율 계산, 배송비 분기 등)을 강제한다.
- **PII 보호:** 개인정보(이름, 전화번호, 주소)는 `bus_rider_private` 테이블에 분리 저장하고 엄격한 RLS(본인/방장)를 적용한다.
- **분쟁 방지:** 삭제(DELETE) 권한은 '이미 돈을 낸 파티원'이 존재할 경우, 혹은 '주문이 시작된 이후'에는 방장이라도 즉시 차단한다.

## 3. 개발 워크플로우
- **생각하고 코딩하기:** 기능을 구현하기 전, 먼저 5가지 항목(PM 목적, 기술적 위험, DB 구조, 배포 전략, 룰 준수)을 분석한다.
- **트랜잭션 최우선:** 2개 이상의 테이블 수정은 무조건 RPC 트랜잭션으로 묶는다.
